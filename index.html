ทดลองใช้งานทดสอบระบบจำลองแบบอินเตอร์แอคทีฟ ข้อมูลจะคืนค่าเดิมเมื่อกด Refresh หน้าเว็บ

เลือกดูหน้าทดสอบระบบจำลองฟังก์ชันการทำงานได้ฟรีทุกระบบ

กำลังโหลดข้อมูลจากคลังระบบ...

ระบบนี้ทำงานอย่างไร? (อธิบายแบบคนทั่วไปเข้าใจ)

เพื่อให้เข้าใจง่าย ลองนึกภาพว่าคุณกำลัง "สร้างบ้านออนไลน์" ครับ

1. GitHub (เปรียบเหมือน "ตู้เซฟเก็บแบบแปลน")

หน้าที่: เป็นที่สำหรับเก็บไฟล์โค้ด (Source Code) ของคุณทั้งหมดไว้บนอินเทอร์เน็ต

ทำไมต้องใช้: เพื่อป้องกันโค้ดหาย และใช้เป็นแหล่งต้นทาง (Source) ในการส่งโค้ดไปให้ระบบอื่นๆ นำไปแสดงผล

2. Vercel (เปรียบเหมือน "ที่ดินและพื้นที่ก่อสร้าง")

หน้าที่: ดึงไฟล์จาก GitHub มา "สร้างบ้าน" (Deploy) ให้คนทั่วไปเข้าดูเว็บได้

โดเมนฟรี (.vercel.app): Vercel จะให้ชื่อเว็บฟรี เช่น my-project.vercel.app ทำให้เว็บเข้าถึงได้จากทั่วโลกโดยไม่ต้องซื้อชื่อโดเมนในช่วงเริ่มต้น

จุดเด่น: เร็วมาก! ทุกครั้งที่คุณแก้ไขโค้ดใน GitHub ระบบของ Vercel จะอัปเดตหน้าเว็บให้คุณโดยอัตโนมัติ (ไม่ต้องกดอัปโหลดไฟล์เองใหม่)

3. Firebase (เปรียบเหมือน "สมองและคลังเก็บของ")

หน้าที่: เป็นตัวจัดการ "ข้อมูล" (Database)

ทำไมต้องแยก: เพราะ Vercel เก็บไฟล์หน้าเว็บได้ แต่เก็บข้อมูลแบบเปลี่ยนแปลงตลอดเวลา (เช่น ประวัติการซื้อ, ข้อมูลต่างๆ) ได้ไม่ดีนัก Firebase จึงเข้ามาทำหน้าที่นี้แทน

การเชื่อมต่อ: หน้าเว็บที่อยู่บน Vercel จะคุยกับ Firebase ตลอดเวลา เพื่อดึงข้อมูลมาแสดง หรือบันทึกข้อมูลใหม่ลงไป

ขั้นตอนการติดตั้งรบบ (สำหรับผู้ใช้ใหม่)

System Installation Guide

1

เตรียมอุปกรณ์ (GitHub & Firebase)

สร้างบ้านให้โค้ด: สมัครสมาชิก GitHub และนำซอร์สโค้ดที่คุณได้รับไปวางไว้ในระบบ เพื่อให้ง่ายต่อการเก็บรักษาและการแก้ไข

สร้างสมองให้ระบบ: สมัครใช้งาน Firebase เพื่อเตรียมฐานข้อมูลที่จะใช้บันทึกข้อมูลสำคัญต่างๆ

2

เชื่อมต่อระบบ (Connect & Deploy)

เชื่อมต่อคลังข้อมูล: นำรหัสเชื่อมต่อ (Config Key) จาก Firebase มาใส่ในไฟล์โค้ดของคุณ เหมือนการเสียบปลั๊กให้ระบบเริ่มทำงาน

นำระบบขึ้นเว็บ: เชื่อมต่อ GitHub เข้ากับ Vercel ระบบจะทำการสร้างหน้าเว็บไซต์ให้คุณทันทีโดยใช้โดเมนฟรี

3

เริ่มใช้งาน (Go Live)

เมื่อติดตั้งเสร็จสิ้น ระบบของคุณจะออนไลน์และทำงานได้ทันทีอย่างรวดเร็ว

ขั้นตอนการสั่งซื้อและรับระบบ

เข้าถึงระบบซอร์สโค้ดพร้อมใช้งานภายใน 3 ขั้นตอนง่ายๆ

1

เลือกและทดสอบระบบ

กดทดลองเล่นแอปพลิเคชันผ่านปุ่มทดสอบระบบ เพื่อทดสอบฟังก์ชันการเพิ่ม ลบ หรือแก้ไขข้อมูลเสมือนระบบจริง

2

สแกนชำระเงิน

กดที่ปุ่มสแกนจ่ายของระบบ Premium เพื่อเปิด QR Code ชำระเงิน จากนั้นสแกนจ่ายผ่านโมบายแบงก์กิ้งได้ทันที

3

โหลดโค้ด

ดาวน์โหลด Source Code เพื่อนำไปใช้งาน

Educational & Sandbox Use

  • เหมาะสำหรับผู้ที่ต้องการทดลองใช้งานระบบ หรือศึกษาโครงสร้างโค้ดเบื้องต้น

  • ผู้ใช้ต้องมีพื้นฐานในการตั้งค่า Firebase และ Hosting ด้วยตนเองจากคู่มือที่จัดเตรียมไว้ให้

  • การทดลองใช้งาน คือระบบใช้งานได้จริง เพียงแต่ยังไม่มีฐานข้อมูล ซึ่งจะเป็นข้อมูลจำลอง เมื่อรีเฟรช ข้อมูลจะกลับมาเป็นค่าเริ่มต้น
